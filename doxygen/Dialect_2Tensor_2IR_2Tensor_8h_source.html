48 /// Determines whether tensor::CastOp casts to a more dynamic version of the
49 /// source tensor. This is useful to fold a tensor.cast into a consuming op and
50 /// implement canonicalization patterns for ops in different dialects that may
51 /// consume the results of tensor.cast operations. Such foldable tensor.cast
52 /// operations are typically inserted as `subtensor` ops and are canonicalized,
53 /// to preserve the type compatibility of their uses.
54 ///
55 /// Returns true when all conditions are met:
56 /// 1. source and result are ranked tensors with same element type and rank.
57 /// 2. the tensor type has more static information than the result
58 ///
59 /// Example:
60 /// ```mlir
61 /// %1 = tensor.cast %0 : tensor<8x16xf32> to tensor<?x?xf32>
62 /// %2 = consumer %1 ... : tensor<?x?xf32> ...
63 /// ```
64 ///
65 /// folds into:
66 ///
67 /// ```mlir
68 /// %2 = consumer %0 ... : tensor<8x16xf32> ...
69 /// ```
70 bool canFoldIntoConsumerOp(CastOp castOp);
71 
72 /// Performs folding of any operand of `op` if it comes from a tensor::CastOp
73 /// that can be folded.
