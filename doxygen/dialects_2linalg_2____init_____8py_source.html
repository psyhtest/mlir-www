10 # These are the ground truth functions defined as:
11 # ```
12 # @linalg_structured_op
13 # def matmul(A=TensorDef(T1, S.M, S.K),
14 # B=TensorDef(T2, S.K, S.N),
15 # C=TensorDef(U, S.M, S.N, output=True)):
16 # ```
17 # using the linalg-py eDSL.
18 # The linalg-py eDSL builds a python representation (PyRepr) that is
19 # used in following ways:
20 # 1. PyRepr -> YAML to generate the C++ and Python .td files. These
21 # then turn into the core C++ Op classes and Python OpView classes
22 # respectively (made available in _linalg_ops_gen). The generic OpView class
23 # mechanism makes the C++ classes available to python through the CAPI.
24 # PyRepr -> YAML currently occurs before compiler compile time.
25 # The other steps in this category occur at compiler compile time.
26 # 2. PyRepr -> linalg.core_named_ops calls: piggybacks on the
27 # _linalg_ops_gen classes and the OpView mechanism to build IR at
28 # runtime in python:
29 # a. by default, the Named Op Form is emitted, e.g.:
30 # `linalg.matmul(lhs, rhs, outs=[out])` creates the following IR:
31 # ```
32 # %1 = linalg.matmul ins(%arg0, %arg1 : tensor<4x16xf32>, tensor<16x8xf32>)
33 # outs(%0 : tensor<4x8xf32>)
34 # -> tensor<4x8xf32>
35 # ```
36 # b. by setting emit_generic=True, the Generic Op Form is emitted, e.g.:
37 # `linalg.matmul(lhs, rhs, outs=[out], emit_generic=True)` creates the following IR:
38 # ```
39 # %1 = linalg.generic {indexing_maps = [...], iterator_types = [...]}
40 # ins(%arg0, %arg1 : tensor<4x16xf32>, tensor<16x8xf32>)
41 # outs(%0 : tensor<4x8xf32>) {
42 # ^bb0(%arg2: f32, %arg3: f32, %arg4: f32):
43 # ...
44 # linalg.yield %3 : f32
45 # } -> tensor<4x8xf32>
46 # ```
47 # 3. PyRepr -> Runtime Custom Op definitions: directly generates a
48 # linalg.generic form like in 2.b.
49 # !!!WARNING!!!: if one creates a runtime custom op with the same name
50 # as an existing core named op, step 2. will likely take precedence.
